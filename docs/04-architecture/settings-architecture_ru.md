# Архитектура настроек
Языки: [English](settings-architecture.md) | [Русский](settings-architecture_ru.md)

## Обзор

Настройки в TgBrain управляются через двухуровневую систему: переменные окружения для начальных значений по умолчанию и база данных для переопределений во время выполнения.

## Уровни настроек

```
.env file --> Settings class --> Database --> Runtime
```

1. **Переменные окружения** -- Начальные значения по умолчанию загружаются из `.env` через pydantic-settings
2. **База данных** -- Постоянные переопределения, хранящиеся в выделенных таблицах настроек
3. **Runtime** -- Активные настройки, используемые сервисами

При запуске значения из базы данных переопределяют значения по умолчанию из переменных окружения.

## Таблицы настроек

| Таблица | Назначение |
|-------|---------|
| `llm_providers` | Конфигурации LLM-провайдеров |
| `embedding_providers` | Конфигурации провайдеров эмбеддингов |
| `chat_settings` | Конфигурация для каждого чата |
| `app_settings` | Настройки уровня приложения |
| `reindex_settings` | Конфигурация сервиса переиндексации |

## Репозитории настроек

Каждая таблица настроек имеет соответствующий класс репозитория:

| Репозиторий | Таблица |
|------------|-------|
| `LLMProvidersRepository` | `llm_providers` |
| `EmbeddingProvidersRepository` | `embedding_providers` |
| `ChatSettingsRepository` | `chat_settings` |
| `AppSettingsRepository` | `app_settings` |
| `PendingCleanupSettingsRepository` | `app_settings` (ключи pending.*) |
| `SummaryCleanupSettingsRepository` | `app_settings` (ключи summary_cleanup.*) |
| `EncryptionSettingsRepository` | `app_settings` (ключи encryption.*) |

## Инициализация настроек

При запуске приложения:

1. Загрузка значений `.env` в класс Settings
2. Инициализация соединения с базой данных
3. Запуск инициализатора настроек, который:
    - Создаёт записи провайдеров в базе данных, если они не существуют
    - Применяет TG_CHAT_ENABLE/TG_CHAT_DISABLE к chat_settings
    - Синхронизирует значения по умолчанию из .env в базу данных (без перезаписи изменений пользователя)

## Обновления во время выполнения

Все изменения настроек, внесённые через API, немедленно сохраняются в базе данных. Сервисы, зависящие от настроек, уведомляются об изменениях:

- **Мониторинг чатов** -- Кеш Ingester инвалидируется при изменении настроек чата
- **LLM-провайдер** -- Активный провайдер перезагружается при изменении
- **Провайдер эмбеддингов** -- Активный провайдер перезагружается при изменении

## Settings API

Все настройки можно управлять через эндпоинты Settings API. Полная документация: [Settings API](../03-api-reference/settings-api_ru.md).
