# Schedule
Языки: [English](schedule.md) | [Русский](schedule_ru.md)

## Обзор

Служба расписания предоставляет cron-подобное планирование для автоматической генерации суммаризаций. Каждый чат может иметь независимое расписание, которое запускает генерацию суммаризации в указанное время.

## Форматы расписаний

### Простой формат времени

Укажите время в формате HH:MM. Суммаризация будет генерироваться ежедневно в это время.

```json
{"schedule": "09:00"}
```

### Формат Cron

Используйте cron-выражения напрямую.

```json
{"schedule": "0 9 * * 1-5"}
```

Этот пример генерирует суммаризации в 9:00 только по будням.

## Часовые пояса

Расписания хранятся в UTC. При указании времени в формате `HH:MM` сервер преобразует его в UTC, используя часовой пояс приложения (настраивается через переменную `TIMEZONE` или настройки приложения).

## Как это работает

1. ScheduleService работает как фоновая задача asyncio
2. Каждую минуту проверяется, какие расписания совпадают с текущим временем UTC
3. Совпадающие расписания запускают генерацию суммаризации для соответствующих чатов

## Управление расписаниями

### Установка расписания

```bash
curl -X PUT http://localhost:8000/api/v1/settings/chats/{chat_id}/summary/schedule \
  -H "Content-Type: application/json" \
  -d '{"schedule": "09:00"}'
```

Время интерпретируется в часовом поясе приложения и конвертируется в UTC для хранения.

### Получение расписания

```bash
curl http://localhost:8000/api/v1/settings/chats/{chat_id}/summary/schedule
```

### Очистка расписания

```bash
curl -X DELETE http://localhost:8000/api/v1/settings/chats/{chat_id}/summary/schedule
```

## Архитектура

```
ScheduleService
  |-- Scheduler           -- основной цикл, проверка каждую минуту
  |-- TimezoneConverter   -- преобразование локального времени в UTC
  |-- TriggerHandler      -- запуск генерации суммаризации
  |-- ScheduleStore       -- загрузка расписаний из базы данных
```
