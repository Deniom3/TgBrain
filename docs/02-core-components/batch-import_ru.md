# Batch Import
Языки: [English](batch-import.md) | [Русский](batch-import_ru.md)

## Обзор

Система пакетного импорта позволяет импортировать исторические сообщения из файлов экспорта Telegram Desktop. Это позволяет создавать базу знаний из существующей истории чата.

## Как это работает

1. **Загрузка** -- отправка JSON-файла экспорта Telegram Desktop
2. **Парсинг** -- файл разбирается с помощью потокового JSON (ijson) для эффективности использования памяти
3. **Обработка** -- сообщения фильтруются, проходят эмбеддинг и сохраняются батчами
4. **Отслеживание** -- прогресс отслеживается и доступен через API

## Импорт сообщений

### Запуск импорта

```bash
curl -X POST http://localhost:8000/api/v1/messages/import \
  -F "file=@result.json" \
  -F "chat_id=-1001234567890"
```

Ответ:

```json
{
  "task_id": "uuid-here",
  "status": "pending",
  "chat_id": -1001234567890,
  "file_name": "result.json"
}
```

### Проверка прогресса

```bash
curl http://localhost:8000/api/v1/messages/import/{task_id}/progress
```

Ответ:

```json
{
  "task_id": "uuid-here",
  "status": "processing",
  "total_messages": 50000,
  "processed_messages": 12500,
  "imported_messages": 11800,
  "skipped_messages": 700,
  "progress_percent": 25.0
}
```

### Отмена импорта

```bash
curl -X DELETE http://localhost:8000/api/v1/messages/import/{task_id}/cancel
```

## Формат экспорта Telegram Desktop

Система ожидает JSON-файлы в формате экспорта Telegram Desktop:

```json
{
  "name": "Название чата",
  "type": "private_channel",
  "id": 1234567890,
  "messages": [
    {
      "id": 1,
      "type": "message",
      "date": "2026-01-01T10:00:00",
      "from": "Имя пользователя",
      "from_id": "user123",
      "text": "Содержимое сообщения",
      "text_entities": [...]
    }
  ]
}
```

## Обработка

Задачи импорта выполняются как фоновые задачи asyncio. Большие файлы обрабатываются батчами для управления использованием памяти. Потоковый JSON-парсер (ijson) позволяет обрабатывать файлы, превышающие доступный объём оперативной памяти.

## Обработка дубликатов

Сообщения с одинаковыми `message_id` и `chat_id` пропускаются во время импорта. Это позволяет повторно запускать импорт без создания дубликатов.

## Управление файлами

Загруженные файлы временно хранятся во время обработки и очищаются после завершения или отмены импорта.
