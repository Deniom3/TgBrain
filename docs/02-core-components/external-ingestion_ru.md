# External Ingestion
Языки: [English](external-ingestion.md) | [Русский](external-ingestion_ru.md)

## Обзор

API внешнего ingestion позволяет отправлять сообщения из внешних источников, таких как пользовательские боты, вебхуки или другие приложения. Сообщения обрабатываются через тот же конвейер, что и сообщения, полученные из Telegram.

## Эндпоинт ingestion

```bash
curl -X POST http://localhost:8000/api/v1/messages/ingest \
  -H "Content-Type: application/json" \
  -d '{
    "chat_id": -1001234567890,
    "text": "Содержимое сообщения",
    "date": "2026-04-01T10:00:00Z",
    "sender_id": 123456,
    "sender_name": "Имя бота",
    "message_link": "https://t.me/c/1234567890/100",
    "is_bot": true,
    "is_action": false
  }'
```

## Поля запроса

| Поле | Тип | Обязательно | Описание |
|-------|------|----------|-------------|
| `chat_id` | integer | Да | ID чата Telegram |
| `text` | string | Да | Текстовое содержимое сообщения |
| `date` | string | Да | Временная метка сообщения (ISO 8601) |
| `sender_id` | integer | Да | ID пользователя или бота-отправителя |
| `sender_name` | string | Да | Отображаемое имя отправителя |
| `message_link` | string | Нет | Ссылка на сообщение |
| `is_bot` | boolean | Нет | Является ли отправитель ботом |
| `is_action` | boolean | Нет | Является ли сообщение системным действием |

## Ответ

```json
{
  "success": true,
  "status": "processed",
  "message_id": 100,
  "chat_id": -1001234567890,
  "filtered": false,
  "pending": false,
  "duplicate": false,
  "updated": false
}
```

Значения статуса: `processed`, `pending`, `filtered`, `duplicate`, `updated`.

## Коды ошибок

| Код | HTTP Status | Описание |
|------|-------------|-------------|
| EXT-001 | 400 | Неверные данные запроса |
| EXT-002 | 400 | Чат не отслеживается |
| EXT-003 | 200 | Ошибка эмбеддинга (сообщение сохранено как отложенное) |
| EXT-004 | 200/500 | Ошибка базы данных |
| EXT-005 | 200 | Отфильтрованное сообщение |
| EXT-006 | 200 | Сообщение-дубликат |
| EXT-007 | 200 | Сервис эмбеддингов недоступен (сообщение сохранено как отложенное) |

## Поток обработки

1. Валидация данных запроса
2. Проверка, отслеживается ли чат (EXT-002, если нет)
3. Проверка на дубликаты сообщений (пропуск эмбеддинга для точных дубликатов)
4. Применение фильтров сообщений
5. Генерация эмбеддинга
6. Сохранение сообщения в базе данных
7. Возврат ответа

## Отложенные сообщения

Если эмбеддинг не удался, но сообщение валидно, оно сохраняется в таблице `pending_messages`. Служба очистки отложенных сообщений попытается обработать его позже.

## Требование мониторинга чата

Целевой чат должен быть зарегистрирован и отслеживаться в системе. Используйте Chat Management API для включения мониторинга перед ingestion сообщений.
