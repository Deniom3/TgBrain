# Telegram Ingestion
Языки: [English](telegram-ingestion.md) | [Русский](telegram-ingestion_ru.md)

## Обзор

Telegram Ingester — это основной компонент сбора сообщений. Он подключается к Telegram через Telethon, опрашивает отслеживаемые чаты на наличие новых сообщений, фильтрует нежелательный контент, генерирует эмбеддинги и сохраняет сообщения в базе данных.

## Архитектура

```
TelegramIngester
  |-- SessionManager          -- жизненный цикл сессии Telegram
  |-- TelegramConnection      -- управление подключением
  |-- MessageProcessing       -- конвейер обработки сообщений
  |-- Filters                 -- фильтрация сообщений (боты, реклама, действия)
  |-- Saver                   -- сохранение в базу данных
  |-- PendingCleanupService   -- очистка устаревших отложенных сообщений
  |-- ChatSyncService         -- синхронизация метаданных чатов
```

## Конвейер обработки сообщений

1. **Опрос** -- получение новых сообщений из отслеживаемых чатов через Telethon
2. **Фильтрация** -- удаление сообщений ботов, рекламы, системных действий и другого нежелательного контента
3. **Эмбеддинг** -- генерация векторных эмбеддингов для текста сообщения
4. **Хранение** -- сохранение сообщения и эмбеддинга в базе данных
5. **Подтверждение** -- обновление смещения последнего обработанного сообщения

## Мониторинг чатов

Чаты отслеживаются на основе настроек в таблице `chat_settings`. Обрабатываются только чаты с `is_monitored = true`.

### Управление отслеживаемыми чатами

Для управления отслеживаемыми чатами используйте Settings API:

```bash
# Список всех чатов
curl http://localhost:8000/api/v1/settings/chats

# Включить мониторинг для чата
curl -X POST http://localhost:8000/api/v1/settings/chats/{chat_id}/enable

# Отключить мониторинг для чата
curl -X POST http://localhost:8000/api/v1/settings/chats/{chat_id}/disable

# Синхронизировать чаты с Telegram
curl -X POST http://localhost:8000/api/v1/settings/chats/sync
```

### Конфигурация при запуске

При запуске чаты можно включить или отключить через переменные окружения:

```env
TG_CHAT_ENABLE=-1001234567890,-1009876543210
TG_CHAT_DISABLE=-1001111111111
```

Эти настройки применяются один раз при запуске. Последующие изменения следует вносить через API.

## Фильтрация сообщений

Конвейер ingest применяет несколько фильтров:

| Фильтр | Описание |
|--------|-------------|
| Сообщения ботов | Сообщения от аккаунтов ботов |
| Рекламные сообщения | Продвигаемые сообщения/посты каналов в группах |
| Системные действия | Сообщения о входе/выходе/закреплении |
| Сервисные сообщения | Изменение фото чата, изменение названия |
| Пустые сообщения | Сообщения без текстового содержимого |

Фильтры можно настраивать для каждого чата через настройки чата.

## Хранение сообщений

Сообщения хранятся в таблице `messages` со следующими полями:

| Поле | Тип | Описание |
|-------|------|-------------|
| `message_id` | bigint | ID сообщения Telegram |
| `chat_id` | bigint | ID чата |
| `sender_id` | bigint | ID пользователя-отправителя |
| `sender_name` | text | Отображаемое имя отправителя |
| `text` | text | Текстовое содержимое сообщения |
| `date` | timestamptz | Временная метка сообщения |
| `embedding` | vector | Векторный эмбеддинг |
| `is_bot` | boolean | Является ли отправитель ботом |
| `is_action` | boolean | Является ли сообщение системным действием |

## Отложенные сообщения

Сообщения, прошедшие начальную проверку, но не pouvant быть немедленно обработаны или сохранены, помещаются в таблицу `pending_messages`. Фоновая служба очистки удаляет устаревшие отложенные сообщения на основе настраиваемых параметров TTL.

Подробности см. в [Pending Cleanup](pending-cleanup_ru.md).

## Жизненный цикл ingestion

Ingester работает как фоновая задача asyncio в течение жизненного цикла приложения:

1. **Запуск** -- загрузка сессии, подключение к Telegram, запуск цикла опроса
2. **Опрос** -- периодическое получение новых сообщений из всех отслеживаемых чатов
3. **Обработка** -- фильтрация, создание эмбеддингов и хранение каждого сообщения
4. **Обработка ошибок** -- повтор при временных сбоях, логирование постоянных сбоев
5. **Остановка** -- корректное закрытие подключения к Telegram, сохранение состояния

## Ограничение скорости

Ingester использует адаптивный ограничитель скорости для избежания ошибок FloodWait API Telegram. Подробности см. в [Rate Limiter](rate-limiter_ru.md).

## Внешние источники сообщений

Сообщения также могут поступать из внешних источников (боты, вебхуки) через эндпоинт `POST /api/v1/messages/ingest`. Подробности см. в [External Ingestion](external-ingestion_ru.md).

## Пакетный импорт

Исторические сообщения можно импортировать из файлов экспорта Telegram Desktop. Подробности см. в [Batch Import](batch-import_ru.md).
