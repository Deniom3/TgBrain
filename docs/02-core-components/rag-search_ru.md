# RAG Search
Языки: [English](rag-search.md) | [Русский](rag-search_ru.md)

## Обзор

RAG (Retrieval-Augmented Generation) обеспечивает семантический поиск по всем сохранённым сообщениям. Пользователи задают вопросы на естественном языке и получают ответы, основанные на реальной истории чата.

## Как это работает

1. **Эмбеддинг запроса** -- вопрос пользователя преобразуется в векторный эмбеддинг
2. **Векторный поиск** -- HNSW-индекс pgvector находит наиболее похожие сообщения
3. **Расширение контекста** -- для контекста извлекаются окружающие сообщения
4. **Ответ LLM** -- вопрос и контекст отправляются в LLM для генерации ответа
5. **Атрибуция источников** -- исходные сообщения возвращаются как источники

## Использование Search API

```bash
curl -X POST http://localhost:8000/api/v1/ask \
  -H "Content-Type: application/json" \
  -d '{
    "query": "Какое решение мы приняли по графику развёртывания?",
    "chat_id": -1001234567890,
    "top_k": 5
  }'
```

Ответ:

```json
{
  "answer": "Судя по истории чата, команда решила развернуть в пятницу...",
  "sources": [
    {
      "message_id": 12345,
      "chat_id": -1001234567890,
      "text": "Давайте развернём в пятницу после завершения тестирования.",
      "sender_name": "Alice",
      "date": "2026-03-15T10:30:00Z",
      "similarity": 0.87
    }
  ]
}
```

## Конфигурация

| Параметр | По умолчанию | Описание |
|-----------|---------|-------------|
| `RAG_TOP_K` | 5 | Количество извлекаемых сообщений |
| `RAG_SCORE_THRESHOLD` | 0.3 | Минимальный показатель схожести |

## Расширение контекста

Расширитель контекста извлекает сообщения, окружающие найденные результаты, чтобы предоставить LLM лучший контекст. Это важно, потому что отдельные сообщения могут не давать полной картины.

## Область поиска

Поиск может быть ограничен конкретными чатами или выполнен по всем отслеживаемым чатам:

```bash
# Поиск в конкретном чате
{"query": "...", "chat_id": -1001234567890}

# Поиск по всем чатам
{"query": "..."}
```

## Поиск по суммаризациям

Система RAG также выполняет поиск по сохранённым суммаризациям, что позволяет отвечать на запросы вроде «Что произошло на прошлой неделе?» на основе предварительно сгенерированных суммаризаций.

## Производительность

HNSW-индекс в pgvector обеспечивает быстрый приблизительный поиск ближайших соседей. Производительность индекса зависит от:

- Размерности вектора (автоопределяется из модели эмбеддинга)
- Количество проиндексированных сообщений
- Параметров HNSW (m, ef_construction, ef_search)

Реиндексация требуется при изменении модели эмбеддинга. Подробности см. в [Reindex](reindex_ru.md).
