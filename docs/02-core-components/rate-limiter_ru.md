# Rate Limiter
Языки: [English](rate-limiter.md) | [Русский](rate-limiter_ru.md)

## Обзор

Адаптивный ограничитель скорости защищает от ошибок FloodWait API Telegram, динамически регулируя частоту запросов на основе ответов сервера.

## Как это работает

1. **Отслеживание запросов** -- каждый запрос API логируется с информацией о времени
2. **Мониторинг ответов** -- ошибки FloodWait перехватываются и записываются
3. **Корректировка скорости** -- ограничитель снижает частоту запросов при возникновении ошибок FloodWait
4. **Постепенное восстановление** -- скорость медленно увеличивается после периода охлаждения
5. **Сохранение истории** -- история инцидентов хранится в базе данных для анализа

## Обработка FloodWait

Когда Telegram возвращает ошибку FloodWait, ограничитель скорости:

1. Записывает инцидент в таблицу `flood_wait_incidents`
2. Приостанавливает запросы на длительность, указанную Telegram
3. Уменьшает размер батча для последующих запросов
4. Логирует инцидент для мониторинга

## Мониторинг

### Просмотр текущей пропускной способности

```bash
curl http://localhost:8000/api/v1/system/throughput
```

### Просмотр истории FloodWait

```bash
curl http://localhost:8000/api/v1/system/flood-history
```

### Просмотр системной статистики

```bash
curl http://localhost:8000/api/v1/system/stats
```

## Конфигурация

Ограничитель скорости самонастраивается и не требует ручной конфигурации. Он адаптируется на основе:

- Частоты недавних запросов
- Частоты инцидентов FloodWait
- Времени с момента последнего инцидента

## Архитектура

```
TelegramRateLimiter
  |-- Отслеживание запросов    -- логирование каждого запроса с временной меткой
  |-- Запись инцидентов  -- хранение событий FloodWait в БД
  |-- Корректировка скорости     -- динамическое изменение размеров батчей
  |-- Управление историей  -- запрос и анализ прошлых инцидентов
```
