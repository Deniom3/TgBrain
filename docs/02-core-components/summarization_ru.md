# Summarization
Языки: [English](summarization.md) | [Русский](summarization_ru.md)

## Обзор

Система суммаризации генерирует дайджесты активности чата с помощью ИИ за настраиваемые периоды времени. Суммаризации генерируются асинхронно и могут быть запланированы для автоматической генерации.

## Как это работает

1. Получен запрос на генерацию суммаризации (через API или расписание)
2. Извлекаются сообщения за указанный период времени
3. Сообщения форматируются и отправляются настроенному LLM-провайдеру
4. Сгенерированная суммаризация сохраняется в таблице `chat_summaries`
5. Для суммаризации автоматически генерируются эмбеддинги
6. Если настроен вебхук, суммаризация доставляется через него

## API генерации суммаризаций

### Генерация для одного чата

```bash
curl -X POST http://localhost:8000/api/v1/chats/{chat_id}/summary/generate \
  -H "Content-Type: application/json" \
  -d '{
    "period_minutes": 1440,
    "use_cache": true
  }'
```

Ответ:

```json
{
  "task_id": "uuid-here",
  "status": "pending",
  "from_cache": false
}
```

### Генерация для всех чатов

```bash
curl -X POST http://localhost:8000/api/v1/chats/summary/generate
```

### Проверка статуса задачи

```bash
curl http://localhost:8000/api/v1/chats/{chat_id}/summary/{task_id}
```

### Получение суммаризаций

```bash
# Список суммаризаций с пагинацией
curl http://localhost:8000/api/v1/chats/{chat_id}/summary?page=1&limit=10

# Получить последнюю суммаризацию
curl http://localhost:8000/api/v1/chats/{chat_id}/summary/latest

# Получить конкретную суммаризацию
curl http://localhost:8000/api/v1/chats/{chat_id}/summary/{summary_id}
```

## Кэширование

Результаты суммаризации кэшируются на основе хэша параметров генерации. TTL кэша зависит от возраста запрашиваемого периода:

| Возраст периода | TTL кэша |
|------------|-----------|
| Менее 24 часов | 2 часа |
| 24-72 часа | 24 часа |
| Более 72 часов | Без истечения срока |

Используйте `use_cache: false` для обхода кэша и принудительной регенерации.

## Расписание

Каждый чат может иметь независимое расписание суммаризации. Расписания поддерживают как простой формат времени, так и cron-выражения.

### Установка расписания

```bash
curl -X PUT http://localhost:8000/api/v1/settings/chats/{chat_id}/summary/schedule \
  -H "Content-Type: application/json" \
  -d '{"schedule": "09:00", "timezone": "Europe/Moscow"}'
```

### Формат Cron

```bash
curl -X PUT http://localhost:8000/api/v1/settings/chats/{chat_id}/summary/schedule \
  -H "Content-Type: application/json" \
  -d '{"schedule": "cron:0 9 * * 1-5", "timezone": "Europe/Moscow"}'
```

Служба расписания работает в фоновом режиме и запускает генерацию суммаризаций в указанное время.

## Пользовательские промпты

Каждый чат может иметь пользовательский промпт для генерации суммаризации:

```bash
curl -X PUT http://localhost:8000/api/v1/settings/chats/{chat_id}/summary/prompt \
  -H "Content-Type: application/json" \
  -d '{"prompt": "Обобщите ключевые технические решения, принятые в этом чате..."}'
```

## Настройки суммаризации

| Настройка | Эндпоинт | Описание |
|---------|----------|-------------|
| Включить/Отключить | `POST /api/v1/settings/chats/{chat_id}/summary/enable` | Переключение генерации суммаризаций |
| Период | `PUT /api/v1/settings/chats/{chat_id}/summary/period` | Установка периода суммаризации в минутах |
| Расписание | `PUT/GET/DELETE /api/v1/settings/chats/{chat_id}/summary/schedule` | Управление расписанием |
| Пользовательский промпт | `PUT/GET/DELETE /api/v1/settings/chats/{chat_id}/summary/prompt` | Управление пользовательским промптом |

## Статистика суммаризаций

```bash
curl http://localhost:8000/api/v1/chats/summary/stats
```

Возвращает количество суммаризаций, время генерации и процент попаданий в кэш.

## Автоматическая очистка

Старые задачи суммаризации автоматически очищаются на основе настраиваемых параметров хранения:

| Настройка | По умолчанию | Описание |
|---------|---------|-------------|
| `pending_timeout_minutes` | 60 | Удалить отложенные задачи по истечении этого времени |
| `processing_timeout_minutes` | 5 | Отметить обрабатываемые задачи как неудачные по истечении этого времени |
| `failed_retention_minutes` | 120 | Удалить неудачные задачи по истечении этого времени |
| `completed_retention_minutes` | None | Завершённые задачи хранятся бессрочно |

## Доставка через вебхук

При генерации суммаризации и наличии настроенного вебхука для чата суммаризация автоматически доставляется. Настройки см. в [Webhook](webhook_ru.md).
