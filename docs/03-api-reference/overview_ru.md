# Обзор API
Языки: [English](overview.md) | [Русский](overview_ru.md)

## Базовый URL

```
http://localhost:8000
```

## Интерактивная документация

Swagger UI доступен по адресу http://localhost:8000/docs для интерактивного изучения API.

## Аутентификация

Большинство эндпоинтов настроек и управления требуют аутентификации. Эндпоинты QR Auth используются для установки Telegram-сессии, которая служит механизмом аутентификации.

## Формат запросов

Все тела запросов используют формат JSON с заголовком `Content-Type: application/json`.

## Формат ответов

Все ответы используют формат JSON. Ответы с ошибками включают код ошибки и сообщение.

### Успешный ответ

```json
{
  "status": "ok",
  "data": { ... }
}
```

### Ответ с ошибкой

```json
{
  "status": "error",
  "code": "EXT-001",
  "message": "Invalid request data: chat_id is required"
}
```

## Группы API

| Группа | Префикс | Описание |
|--------|---------|----------|
| Health | `/health` | Проверки работоспособности системы |
| RAG Search | `/api/v1/ask` | Семантический поиск |
| Chat Summary | `/api/v1/chats/{chat_id}/summary` | Генерация и получение сводок |
| Message Ingestion | `/api/v1/messages` | Внешний ingest и пакетный импорт |
| Settings | `/api/v1/settings` | Все эндпоинты конфигурации |
| System | `/api/v1/system` | Мониторинг и статистика |

## Пагинация

Эндпоинты списков поддерживают пагинацию через параметры запроса:

| Параметр | По умолчанию | Описание |
|----------|--------------|----------|
| `page` | 1 | Номер страницы |
| `limit` | 20 | Элементов на странице |

## Ограничение частоты запросов

Эндпоинты API могут быть ограничены по частоте запросов. Проверьте заголовки ответов для получения информации об ограничениях.
