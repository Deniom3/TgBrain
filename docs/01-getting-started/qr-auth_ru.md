# QR-аутентификация
Языки: [English](qr-auth.md) | [Русский](qr-auth_ru.md)

## Обзор

TgBrain использует QR-код аутентификацию для подключения к Telegram. Этот метод не требует ввода номера телефона или получения SMS-кодов — просто отсканируйте QR-код с помощью мобильного приложения Telegram.

## Как это работает

1. Приложение генерирует QR-код, привязанный к временной сессии Telegram
2. Вы сканируете QR-код через Telegram на телефоне
3. Telegram подтверждает вход на вашем телефоне
4. Приложение получает данные сессии и сохраняет их в зашифрованном виде в базе данных
5. При последующих запусках используется сохранённая сессия автоматически

## Аутентификация

### Через веб-интерфейс

1. Запустите приложение
2. Откройте http://localhost:8000/qr-auth в браузере
3. Отобразится QR-код
4. Откройте Telegram на телефоне
5. Перейдите в Settings > Devices > Link Desktop Device
6. Отсканируйте QR-код
7. Подтвердите вход на телефоне

### Через API

Создайте сессию QR-аутентификации:

```bash
curl -X POST http://localhost:8000/api/v1/settings/telegram/qr-code
```

Ответ:

```json
{
  "session_id": "abc123",
  "qr_url": "http://localhost:8000/qr-auth?session=abc123"
}
```

Откройте `qr_url` в браузере, чтобы увидеть QR-код.

Проверка статуса аутентификации:

```bash
curl http://localhost:8000/api/v1/settings/telegram/qr-status/abc123
```

Ответ в режиме ожидания:

```json
{
  "status": "waiting",
  "session_id": "abc123"
}
```

Ответ после успешной аутентификации:

```json
{
  "status": "authenticated",
  "session_id": "abc123"
}
```

Отмена сессии:

```bash
curl -X POST http://localhost:8000/api/v1/settings/telegram/qr-cancel/abc123
```

## Управление сессиями

### Проверка статуса аутентификации

```bash
curl http://localhost:8000/api/v1/settings/telegram/auth-status
```

Ответ:

```json
{
  "authenticated": true,
  "user_id": 123456789,
  "username": "your_username",
  "phone": "+1234567890"
}
```

### Проверка здоровья сессии

```bash
curl http://localhost:8000/api/v1/settings/telegram/check
```

Ответ:

```json
{
  "status": "healthy",
  "user_id": 123456789
}
```

### Выход

```bash
curl -X POST http://localhost:8000/api/v1/settings/telegram/logout
```

Эта команда уничтожает текущую сессию Telegram и удаляет сохранённые данные сессии.

## Хранение сессии

Данные сессии шифруются с помощью библиотеки `cryptography` и хранятся в таблице базы данных `telegram_auth`. Временные файлы сессий создаются во время аутентификации и удаляются после её завершения.

## Автоматическое переподключение

При запуске приложение проверяет наличие сохранённых данных сессии. Если существует валидная сессия, оно автоматически подключается к Telegram и начинает приём сообщений. После начальной настройки ручная аутентификация не требуется.

## Устранение неполадок

### Истечение срока действия QR-кода

QR-коды имеют ограниченное время жизни. Если код истёк, создайте новую сессию через API или обновите веб-страницу.

### Ошибка аутентификации

Убедитесь, что:
- Ваше приложение Telegram обновлено
- Вы сканируете код с правильным аккаунтом Telegram
- Ваше интернет-соединение стабильно

### Сессия недействительна

Если Telegram аннулирует вашу сессию (например, при смене пароля или изменении настроек безопасности), потребуется повторная аутентификация. Проверьте эндпоинт статуса аутентификации и выполните повторную аутентификацию при необходимости.
